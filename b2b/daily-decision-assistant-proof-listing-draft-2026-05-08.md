## 掲載タイトル候補

AI秘書の「今日やること」判断支援 Proof

## 1行説明

予定・タスク・ブロッカーをサンプルデータで整理し、今日やること / 後回し / 着手禁止を分けて、外部アクションは確認キューに回すAI秘書Proofです。

## 掲載カード本文案

AI秘書をいきなり自動実行させるのではなく、まず「何を今日扱うべきか」を整理するための公開Proofです。予定、締切、作業価値、ブロッカーをスコア化し、今日の優先タスク・後回しタスク・着手禁止タスクに分けます。メール送信や予定変更など外部に影響する操作は自動実行せず、確認キューに入れて人が承認する前提にしています。

## B2B向け訴求ポイント

- AIに実行させる前に、判断・整理・確認キューを分離できる
- 完全自動化ではなく、人間承認を前提にした安全な業務導入を説明できる
- 予定、問い合わせ、タスク、社内依頼など複数の情報を日次判断にまとめる型として使える
- 公開repoはサンプルデータのみで、実案件・顧客情報・内部ナレッジを含まない
- CIで公開境界チェックを回し、個人用/内部用の語が混ざるリスクを抑えている

## 成果指標候補

- 朝の確認対象を「今日やる / 後回し / 着手禁止」に分けられるか
- 外部アクションが勝手に実行されず、確認キューに入るか
- 判断理由がスコア・理由・次アクションとして説明できるか
- サンプルデータだけで安全にデモできるか
- public境界チェックにより、内部情報の混入を検知できるか

## 担当範囲として見せる内容

- AI秘書ワークフローの安全境界設計
- サンプルデータ設計
- 優先度スコアリング設計
- confirmation queue 設計
- no-send / no-modify / no-execute の運用ガード
- public proof化に向けたREADME、CI、release整備

## 掲載時の注意

- 「自動で予定変更する」「自動でタスクを実行する」とは書かない
- 「判断支援」「確認キュー」「人間承認前提」の表現に寄せる
- 実運用データではなく、公開用サンプルデータであることを明記する
- B2B本番UIへ載せる場合は、desktop / mobile / 実URLで目視確認してから反映する

## 短縮版コピー

AI秘書が予定やタスクを勝手に実行するのではなく、まず「今日やること」「後回し」「着手禁止」を整理し、外部アクションは確認キューへ回す公開Proofです。サンプルデータのみで動作し、no-send / no-modify / no-execute を前提にした安全な業務導入パターンを示しています。

## CTA近くに置く補足文案

AI導入で重要なのは、最初から完全自動化することではなく、AIに任せる判断支援と人が確認すべき操作を分けることです。このProofでは、外部に影響する操作を確認キューへ分離し、業務で使いやすい安全境界を示しています。

## 次にUIへ載せる場合の最小手順

4. 実装後、local と production URL の両方を目視確認する。
5. 問い合わせCTAへの導線が過剰にならないよう、リンクは1箇所に絞る。
