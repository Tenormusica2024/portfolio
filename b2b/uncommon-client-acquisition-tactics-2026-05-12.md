# 非定番の個人受託獲得手法メモ

作成日: 2026-05-12  
対象: Ezlize / B2Bポートフォリオの販路拡大  
前提: 低〜中単価の小規模AI業務ツール、問い合わせ整理、Gmail/Sheets連携、Web制作周辺の実装受託

## 背景

B2Bポートフォリオは整ってきているが、公開して待つだけでは相談が来にくい。
理由は、発注意欲のある人の導線上にまだ十分置けていないため。

このメモでは、一般的な「SNS運用」「ブログSEO」「クラウドソーシング登録」だけでなく、知人形成なしでも個人で受託につながる可能性がある、やや非定番の導線を整理する。

## 基本判断

B2Bポートフォリオは、単体で受注を生むサイトというより、営業・提案・プロフィール・受注サイトから見せる **信用補助資料** として使う。

重要なのは、次の問い。

- 誰がすでに発注者に近い場所にいるか
- どの業務課題なら、相手が自分の困りごととして認識しているか
- どの入口なら、電話営業や強い対面営業を避けても接点を作れるか

## 1. パートナー営業 / overflow / white-label 狙い

### 概要

エンド顧客へ直接営業するのではなく、既に顧客を持っている支援会社の外部実装担当として入る。

対象候補:

- 小規模Web制作会社
- 士業事務所 / 士業向け支援会社
- 事務代行会社
- 小規模マーケティング支援会社
- ノーコード制作会社
- Webサイト制作をしている個人・小規模チーム

### 提案例

> 御社の既存顧客から派生する、問い合わせフォーム後の管理、Gmail返信下書き、Google Sheets連携、見積書・発注書の自動生成などを、外部実装担当としてスポット対応できます。

### 狙い

- 相手は既に顧客を持っている
- 顧客の業務課題を聞く立場にいる
- ただしAI/GAS/小規模自動化まで自社で対応できないことがある
- 自社ブランドの裏側で使える外部実装者は需要がある

### メリット

- 直エンド営業より、発注意欲のある案件に近い
- 電話営業・飛び込みを避けやすい
- B2Bポートフォリオを「実装力の証拠」として使いやすい
- 協業提案なので売り込み感を下げられる

### 注意

- 「顧客を奪わない」ことを明示する
- 最初から大きな契約を狙わず、スポット対応・裏方対応にする
- NDA対応可、白ラベル対応可、社名を出さない対応可を整理しておく

## 2. 公開されている困りごとへの1枚診断営業

### 概要

普通の営業メールではなく、相手の公開サイトや問い合わせ導線を軽く見て、1枚の改善メモとして送る。

見る観点:

- 問い合わせフォーム後の管理が弱そう
- 予約フォームがない
- Googleフォームだけで止まっていそう
- 問い合わせ後の返信導線が曖昧
- FAQや料金表がなく、営業対応が重そう
- 資料請求後の自動返信や台帳化がなさそう

### 提案例

> サイトを拝見し、問い合わせ後の管理を少し軽くできそうな点を1つだけメモしました。営業メールというより、もし今後フォーム後の管理や返信下書きを整える機会があれば参考になるかもしれません。

### 狙い

- 完全な汎用メールより、相手専用の具体性が出る
- 「AIツールを作りませんか」より、相手の現実の作業に近い
- 1日3〜5件程度でもよい

### 注意

- セキュリティ診断っぽくしない
- 「問題があります」と断定しない
- 脅し文句にしない
- 添付ファイルは警戒されやすいので、本文内に短く書くか、公開URLにする

## 3. 既存SaaSの接続係として売る

### 概要

「AI業務ツール開発」ではなく、既存SaaSや日常ツールの接続・台帳化・返信下書きとして見せる。

対象ツール:

- Gmail
- Googleフォーム
- Google Sheets
- LINE
- Slack
- Chatwork
- Notion
- Airtable
- WordPress
- Formspree
- Make / Zapier / n8n

### 訴求例

- Gmailとスプレッドシートをつないで問い合わせ台帳化します
- Googleフォーム回答から返信下書きを作ります
- LINE通知とSheets記録をつなぎます
- 問い合わせフォーム後の対応漏れを減らします
- 見積書・発注書の作成をSheetsから半自動化します

### 狙い

発注者は最初から「AIツールを作りたい」とは言わないことが多い。
実際には、メール、フォーム、転記、見積、日報、予約管理のような既存作業で困っている。

そのため、ツール名や作業名を前面に出す方が見つけられやすい。

### Ezlizeとの相性

- GAS / Python / Sheets / Gmail / LLM API の組み合わせと相性がよい
- B2Bポートフォリオの既存訴求と矛盾しない
- 低〜中単価の小規模案件に切りやすい

## 4. 求人票を営業リストとして読む

### 概要

求人票を「採用情報」ではなく「業務課題の公開情報」として読む。

狙う求人:

- 営業事務
- カスタマーサポート
- 社内DX
- 情シス補助
- Web担当
- AI推進
- 業務改善
- 事務効率化
- マーケアシスタント

### 提案例

> 採用までのつなぎや、既存担当者の作業負荷軽減として、問い合わせ整理・Gmail/Sheets連携・見積書作成の一部だけスポットで対応できます。

### 狙い

求人を出している会社は、すでにその業務が足りていない可能性がある。
正社員採用と競合するのではなく、採用までの暫定支援・業務整理・小規模自動化として提案する。

### 注意

- 採用担当への大量送信は避ける
- 「採用しなくてよい」という言い方にしない
- 採用の代替ではなく、採用までの暫定支援・作業整理として提案する
- 1社ずつ求人文面に合わせて内容を変える

## 5. 公共・自治体の小規模調達を見る

### 概要

自治体・官公庁・外郭団体の小規模な業務委託、公募、調達情報を定期的に見る。

対象例:

- Webサイト改修
- フォーム整備
- 業務効率化支援
- データ整理
- 研修資料作成
- 小規模システム改修
- 生成AI活用支援

### 狙い

知人形成なしで案件化できる可能性がある。
ただし書類・資格・納税証明・入札参加資格が重い場合がある。

### 運用

- 今すぐ本命にしない
- 月1回のウォッチ程度
- 少額随意契約、見積合わせ、公募型プロポーザルの中で小規模案件があれば拾う

### 注意

- 仕様書作成・提出書類が重い
- 支払いサイトが長い可能性
- 実績要件がある場合は不利
- 最初から大きい案件は狙わない

## 6. GitHub issue / OSS bounty / 小タスク市場から入る

### 概要

英語圏寄りだが、GitHub issue、OSS bounty、小さなバグ修正、SaaSの小改善から入って継続案件化を狙う。

対象例:

- GitHub issue bounty
- OSSスポンサー型の小修正
- 開発者向けツールの小改善
- SaaSのドキュメント・小機能修正

### 狙い

- 知人形成なしで実力を見せやすい
- GitHub実績として残りやすい
- テキスト中心で完結する可能性がある

### 注意

- 英語会話前提は避ける
- テキスト完結の小タスク限定
- 単価は安い可能性がある
- 国内B2B受託の主軸にはしない

## 7. 既存の外部プロフィールを“営業導線”にする

### 概要

LinkedIn、GitHub、Zenn、ミツモア、ココナラなどのプロフィールを、単なる名刺ではなくB2Bポートフォリオへの導線として整える。

実施済み / 方針:

- LinkedIn / GitHub / Zenn / ミツモアはUTM付きURLへ差し替え済み
- ココナラは実名B2B誘導との相性を慎重判断
- メール署名は必要なら追加

### 狙い

B2Bポートフォリオの直接検索流入を待たず、既に見られる場所から流す。

## 優先順位

現時点の優先順位:

1. パートナー営業
2. 既存SaaSの接続係として売る
3. 1枚診断つきフォーム営業
4. 求人票ベースの課題営業
5. 公共・自治体の小規模調達ウォッチ
6. GitHub bounty / 英語圏小タスク

## 次に具体化するタスク

### パートナー営業

- 対象リストを20件作る
- Web制作会社 / 士業 / 事務代行会社で分ける
- 初回メール文を3パターン作る
- B2BポートフォリオURLをUTM付きにする
- 返信率と反応を記録する

### 1枚診断営業

- 診断テンプレを作る
- 1社あたり3分で見られる項目に絞る
- 脅しや粗探しに見えない文面にする

### 既存SaaS接続係

- `Gmail + Sheets`
- `Googleフォーム + Sheets`
- `問い合わせフォーム + 返信下書き`
- `見積書 / 発注書自動生成`

の4つを、B2Bサイト内または営業メールで説明しやすい短文にする。

## Grok4cxへ投げる時の確認観点

次にGrok4cxへ確認する場合は、以下を投げる。

> 個人の小規模フリーランスエンジニアが、知人紹介なしでB2Bの小規模AI業務ツール・Gmail/Sheets連携・問い合わせ整理・Web制作周辺の案件を取るための、一般的なSNS/SEO/クラウドソーシング以外の非定番手法を探しています。既に検討済みの候補は、パートナー営業、制作会社のoverflow/white-label、1枚診断営業、既存SaaSの接続係、求人票ベース営業、公共・自治体小規模調達、GitHub bountyです。これらを上回る、または盲点になりやすい受注導線を、実例・リスク・日本での実行可能性つきで調査してください。

## Grok4cx追加調査結果 2026-05-12

### 結論

Grok4cxの追加調査では、既存候補を上回る、または盲点になりやすい導線として、次の2系統が特に有望だった。

1. **公的・準公的なDX相談導線に入り込む**
   - 商工会議所 / 商工会
   - IT導入補助金の支援事業者 / コンソーシアム
   - 中小企業支援機関、自治体、中小機構系の相談・セミナー導線
2. **需要が発生した瞬間を監視して、テキストで個別提案する**
   - Google Alert
   - IR / 決算 / 採用ページ
   - 新規法人設立情報
   - 既存SaaS導入済み企業の接続・運用課題

待ちの問い合わせや一般的なSNS/SEOより、発注意欲・課題顕在化・相談窓口に近い場所へB2Bポートフォリオを持ち込む方が、現状のEzlizeには合う。

## 方針更新

今後の販路開拓では、次の順で進める。

1. 既存プロフィール整備や受注サイト登録など、無料でできる導線整備
2. 商工会議所・商工会・支援機関など、公的/準公的な相談導線への接触
3. パートナー営業 / 制作会社 overflow / white-label
4. Google Alert / 求人票 / IR / 新規法人情報を使った低頻度の個別提案
5. 有料営業手法や有料資料請求型サービスは、無料導線を一通り試してから検討

比較ビズなど資料請求・有料導線の可能性がある媒体は、今すぐ本命にしない。まずはココナラ/ミツモア/LinkedIn/GitHub/Zenn/商工会議所系など、無料または低コストで試せる導線を優先する。
